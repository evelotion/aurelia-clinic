Aurelia Clinic

Premium Beauty Clinic Management System with Next.js 14


Created by: Indra Dwi Ananda

Version: 1.0.0

1. Introduction

Thank you for purchasing Aurelia Clinic. This is a complete, production-ready Full-Stack Web Application designed specifically for beauty clinics, dermatologists, and aesthetic centers.

Built with the modern tech stack (Next.js 14 App Router, TypeScript, Tailwind CSS v4, and Prisma ORM), it offers a seamless experience for both clinic administrators and patients.

2. Key Features

  • Smart Appointment Booking: Prevent double-booking with our custom scheduling engine.
  • Multi-Branch Support: Manage multiple clinic locations efficiently.
  • Dynamic Pricing: Set different prices for treatments based on the selected branch.
  • Role-Based Access Control (RBAC): Secure dashboards for Admin, Doctors, and Patients.
  • Payment Integration: Ready-to-use Midtrans Payment Gateway integration.
  • Financial Reporting: Interactive revenue charts and downloadable CSV reports.
  • Before-After Gallery: Interactive slider to showcase treatment results.

3. Server Requirements

Before installing the application, ensure your local machine or server meets the following requirements:

  • Node.js: Version 18.x or higher
  • Package Manager: npm (comes with Node.js)
  • Database: PostgreSQL (Local or Cloud like Supabase/Neon)

4. Installation Guide

Follow these steps to get the application running on your local machine:

Step 1: Extract Files

Extract the downloaded ZIP file and open the Main_Files/aurelia-web folder in your code editor (e.g., VS Code).

Step 2: Install Dependencies

Open your terminal within the project directory and run:

npm install

Step 3: Environment Variables

Rename the .env.example file to .env and fill in your configuration details:

# Database
DATABASE_URL="postgresql://user:password@localhost:5432/aureliaclinic"

# NextAuth
NEXTAUTH_SECRET="your-super-secret-key"
NEXTAUTH_URL="http://localhost:3000"

# Midtrans Payment Gateway
NEXT_PUBLIC_MIDTRANS_CLIENT_KEY="your-client-key"
MIDTRANS_SERVER_KEY="your-server-key"

# Cloudinary (Image Storage)
NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME="your-cloud-name"
CLOUDINARY_API_KEY="your-api-key"
CLOUDINARY_API_SECRET="your-api-secret"

# SMTP Email
SMTP_HOST="smtp.gmail.com"
SMTP_PORT=587
SMTP_USER="your-email@gmail.com"
SMTP_PASS="your-app-password"
Note: You can generate a random string for NEXTAUTH_SECRET using openssl rand -base64 32 in your terminal.

5. Database Setup

We use Prisma ORM. Once your database is ready and connected via the DATABASE_URL in your .env file, run the following commands:

Push Schema to Database:

npx prisma db push

Seed Initial Data (Important):

This command will create the default Admin, Doctor, and Patient accounts, as well as dummy branches and treatments.

npx prisma db seed

Run the Application:

npm run dev

Your application is now running at http://localhost:3000.

7. Third-Party APIs Setup

Midtrans (Payment)

Register at Midtrans. Get your Sandbox/Production keys and place them in the .env file. Set your Midtrans Webhook URL to: https://yourdomain.com/api/webhooks/midtrans.

Cloudinary (Image Storage)

Register at Cloudinary. Go to Dashboard to get your Cloud Name, API Key, and API Secret. This is used for uploading Gallery before-after pictures.
